모양은 팩토리 패턴과 비슷하나, 이를 사용하는 client에 더 초점이 맞춰져있다.

팩토리 메서드 패턴 vs 추상 팩토리 패턴
보는 관점의 차이
패토리 메서드 패턴: 구체적 구현체들을(Products) 만드는 concreteCreator 클래스를 숨기고, Creator(팩토리)를 제공 (팩토리를 구현하는 관점)
구체적 객체 생성 과정을 하위 또는 구체적인 클래스로 옮기는 것이 목적
추상 팩토리 패턴: Client 입장에서 Factory를 통해 추상화된 인터페이스만 사용 할 수 있기때문에 concreteCreator를 직접 사용X (팩토리를 사용하는 관점)
여러 객체를 구체적인 클래스에 의존하지 않고 만들 수 있게 하는 것이 목적
